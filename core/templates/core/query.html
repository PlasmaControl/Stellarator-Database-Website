To make a query on the database, the user should first choose a table to search. This will also specify which parameters the user can put condition on. So, for example, the user can search in "desc_runs" table with "descrunid > 100" condition, and select to see other columns which are related to "desc_runs" table. However, the user cannot make a direct search based on "NFP > 5" in "desc_runs" table, the user must use "configurations" or "devices" table for that. The user can check available columns of each table from "Search Column" box.
"Search Column" and "Search Criteria" boxes are optional in case the user only wants to see the results that satisfy given condition. The database operates using SQL and the conditions must be given in SQL syntax. For example, for numeric values, the user can ask values greater than 2 by typing ">2" without double quotes. Similarly, smaller than 2 is "<2", equal to 2 is "=2", greater than or equal to 2 is ">=" and smaller than or equal to 2 is "<=2". In case of text values, for example if the user wants to see the results that are axisymmetric, choose "configurations" table, select "class" column and ask condition " ='AS' ", again without double quotes. Using "=" sign is case sensitive, to find results which include "precise" in them, one can search in "configurations" table, "name" column and ask " LIKE '%precise%' ". For more details about SQL query, refer to web or ask AI :)
"Columns to Show" includes not only the columns of the selected table but all the data which is linked to that table. For example, "desc_runs" and "configurations" tables are connected through configuration name. "desc_runs" and "publications" are linked through publicationid data. So, the user can look for publications related to "NFP > 1" by selecting "configurations" table, "NFP" column, " >1 " condition and author name, citation, doi as the output columns.
